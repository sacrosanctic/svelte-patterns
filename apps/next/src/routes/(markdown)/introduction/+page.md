:::info
The missing documentation for Svelte and SvelteKit.
:::

Consider this docsite an extension of [svelte.dev](https://svelte.dev). As a matter of fact, it is so much it shadows the official docsite. For example, say you want to know more about scoped styles, you can take the url, `https://svelte.dev/docs/svelte/scoped-styles`, add `patterns` to it, `https://sveltepatterns.dev/docs/svelte/scoped-styles`. And you'll be redirect to a page with more information.

So this isn’t a replacement for the official docs — it’s what the official docs _don’t_ cover.

- Logical groupings of concepts
- Expanded examples and REPL
- showcasing how different APIs interact together
- Real-world usage patterns

The Svelte Language exposes low-level APIs that gives developers a lot of flexibility — but that also means it’s easy to get lost or reinvent the wheel. This docsite aims to fill that gap by showcasing common patterns, best practices, and reusable abstractions.

In practice, some APIs — like $effect, bind:, and streaming with promises — tend to introduce complexity or unpredictable behaviour, even though they’re presented equally in the official docs. Here, we highlight those distinctions and explain _why_ some approaches work better.

Enjoy, and if you have an amazing pattern you discovered, please contribute and let the community know!
